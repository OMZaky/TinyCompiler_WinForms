A robust, multi-phase compiler implementation for the extended TINY programming language, engineered entirely in C#. This project is developed as part of the Semester 4 Compiler Construction coursework.

## 📖 About the TINY Language
TINY is a pedagogical programming language designed for learning compiler architecture. This specific implementation supports an extended 31-rule grammar, featuring:
* **Data Types:** `int`, `float`, `string`
* **Control Structures:** `if-elseif-else-end`, `repeat-until`
* **Functions:** Full support for function declarations, parameters, and a required `main()` entry point.
* **Operators:** Standard arithmetic, assignment (`:=`), conditional (`<>`, `<=`, `>=`, etc.), and boolean (`&&`, `||`) operators.
* **Comments:** Multi-line block comments (`/* ... */`) and extended support for single-line comments (`//`).

## 🏗️ Compiler Architecture Pipeline

The compiler is being built in distinct phases, translating raw source code through a standard processing pipeline, integrated with a Windows Forms graphical interface for easy token and symbol table visualization.

### Phase 1: Lexical Analysis (Scanner) — ✅ *Completed*
The Scanner acts as the front-end processor. It reads the raw TINY source code character by character and groups them into meaningful, categorized `Tokens`.
* **Engine:** Custom-built, manual deterministic state machine (character-by-character scanner), strictly avoiding built-in Regex libraries for core tokenization to meet advanced architectural constraints.
* **Key Features:**
  * **Lookahead Logic:** Safely groups and parses 2-character operators (like `:=`, `<=`, and `&&`) without fragmenting them.
  * **Defensive EOF Handling:** Immune to `IndexOutOfRangeException` crashes. Gracefully handles unclosed strings (`"`) and unclosed multi-line comments (`/*`).
  * **Maximal Munch Error Catching:** Groups invalid character sequences (e.g., multiple decimals like `3.11.23`) into a single, cohesive lexical error rather than splitting them into false positives.
  * **Precision Line Tracking:** Accurately syncs line numbers for error reporting, even when navigating through multi-line comments and strings.
  * **O(1) Keyword Validation:** Efficient Dictionary-based separation of generic identifiers from reserved keywords.

### Phase 2: Syntax Analysis (Parser) — 🚧 *In Progress*
The Parser will consume the token stream generated by the Scanner and verify it against the TINY Context-Free Grammar (CFG).
* Generates an Abstract Syntax Tree (AST).
* Catches and reports structural syntax errors (e.g., *Syntax Error: Missing ';' on Line 15*).

### Phase 3: Semantic Analysis & Code Generation — 📅 *Upcoming*
* **Symbol Table:** Type checking and scope validation for functions and variables.
* **Target Code:** Translating the validated AST into intermediate representation or target machine code.

## 🚀 Getting Started

### Prerequisites
* [.NET SDK](https://dotnet.microsoft.com/download) (Version 6.0 or higher recommended)
* Visual Studio 2022 (Recommended for Windows Forms support)

### Installation & Execution
1. Clone the repository:
   ```bash
   git clone [https://github.com/OMZaky/TinyCompiler_WinForms.git](https://github.com/OMZaky/TinyCompiler_WinForms.git)

2. Open the .sln or .csproj file in Visual Studio.
3. Set the project as the Startup Object and click Start (F5) to launch the graphical compiler interface.
4. Paste any TINY source code into the editor and click Compile to view the generated Token Stream and Error Log.
